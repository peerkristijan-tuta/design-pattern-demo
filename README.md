Due to the sheer number of design patterns in this program, creating a class diagram which depicts the program in its totality would be a forbidding task. As such, I have contented myself with merely depicting what I deem to be the most complex or one of the more complex patterns from each of the major pattern types, namely factory method from creational patterns, decorator from structural patterns, and mediator from behavioral patterns. 

1. Execution instructions

Run this program with a single command line argument to select the design pattern to employ. Example: "java DesignPatterns singleton". Here are alternative command line arguments to use: factoryMethod, composite, decorator, adapter, proxy, facade, chainOfResponsibility, state, mediator, command, and observer.

2. Program information

Each design pattern produces a different output. Each output is produced through the design pattern that produced it, but that is not clear from the output themselves as they are the product rather than the process. Altogether in this program, there are 13 classes, one for main and 12 for each of the 12 design patterns. Each design pattern class only contain the method execute(), which instantiates inner classes that are supposed to enact the design patterns.
