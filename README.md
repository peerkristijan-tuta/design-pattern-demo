Kindly consult the class diagram for a visualization of the design patterns in this program. 

1. Execution instructions

Run this program with a single command line argument to select the design pattern to employ. Example: "java DesignPatterns singleton". Here are alternative command line arguments to use: singleton, factoryMethod, composite, decorator, adapter, proxy, facade, chainOfResponsibility, state, mediator, command, and observer.

2. Program information

Each design pattern produces a different output. Each output is produced through the design pattern that produced it, but that is not clear from the output themselves as they are the product rather than the process.
